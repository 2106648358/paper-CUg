## Context

论文基于2024年HiMCM大赛优秀论文中的AHP-EWM混合模型，构建奥运项目评估与预测系统。现有工作（bak/paper_workbench_hym/）已完成Python计算引擎（AHP/EWM/Hybrid）、React前端原型、以及LaTeX论文前三章。主要缺口是：Ch5-Ch6为占位符、HiMCM 76运动数据未融入核心实验、多文件结构需要统一。

约束条件：
- 使用CUGThesis.cls模板（地大武汉）
- XeLaTeX编译，支持中文（xeCJK）
- 单文件tex，便于管理和编译
- 论文语言为中文
- 不需要前端可视化网站章节

## Goals / Non-Goals

**Goals:**
- 创建一个完整的单文件 thesis.tex（约500-800行），覆盖全部六章
- 将HiMCM 76运动百年数据作为核心实验数据源
- 补充性别平等和安全维度的外部数据（Olympedia + PubMed）
- 生成论文所需的全部图表（通过Python脚本）
- 保证XeLaTeX编译通过

**Non-Goals:**
- 不包含前端Web可视化系统的实现细节
- 不包含附录中代码清单的完整粘贴
- 不重写已有的Python计算脚本（直接复用bak中的）
- 不进行大规模数据爬取（以已有数据为主，少量补充）

## Decisions

### 决策1：单文件 vs 多文件结构
- **选择**：单文件 thesis.tex
- **理由**：简化编译流程、避免路径依赖问题、便于版本控制
- **替代方案**：原有main.tex+chapters/结构 — 增加文件管理复杂度

### 决策2：论文叙事主线
- **选择**：数据驱动叙事（HiMCM数据 → EWM客观权重 + IOC政策 → AHP主观权重 → 混合模型 → 预测）
- **理由**：HiMCM是论文题目的核心数据来源，将HiMCM置于核心位置更有说服力
- **替代方案**：方法驱动叙事（先介绍AHP-EWM再套用数据）— 与已有论文同质化

### 决策3：数据分层策略
- **选择**：双层结构 — 12运动（完整六维）做主实验 + 76运动（四维）做辅助分析
- **理由**：性别平等和安全数据需要外部补充，短期内对76运动全覆盖不现实
- **替代方案**：只做12运动 — 数据量小；对76运动全补六维 — 工作量过大

### 决策4：图表生成策略
- **选择**：先手工构造数据+Python脚本生成，论文写好后再补图表
- **理由**：写作优先，图表可以后续补充；visualization.py已有模板
- **替代方案**：先跑通全部代码再写论文 — 写作节奏被打断

## Risks / Trade-offs

- **风险**：性别平等和安全数据对76运动覆盖率不足 → 在论文中明确标注数据局限性，主实验仍用12运动
- **风险**：HiMCM数据中flag football等新项目设项数为0 → 对这类项目采用定性补充评分
- **风险**：CUGThesis.cls与新版ctex的兼容性 → 使用模板自带的字体文件（simhei.ttf等）
- **权衡**：单文件tex可读性降低 → 在文件内用清晰注释分隔章节
