## 二、致命硬伤（逻辑与学术红线）

### 1. 全文核心矛盾——验证逻辑自毁

论文声称模型排名"与实际奥运决策高度一致"（`thesis.tex:52-53`），以此为模型的有效性背书。但排名第一的项目是**电子竞技（0.772分）**，该项目从未进入奥运会。一个被验证为"与实际奥运决策高度一致"的模型，其最高评分却授予了一个从未被奥运实际决策选中过的项目——这叫什么验证？

如果要声称模型有效，则电子竞技排名第一说明模型失灵；如果要力挺电子竞技入选，就不能用"与实际奥运决策高度一致"作为论证。**全文在这个矛盾上不自洽。**

### 2. "两张皮"——AHP/EWM理论堆砌与奥运评估完全脱节

第二章用近300行篇幅（`thesis.tex:280-556`）重复AHP和EWM的教科书级公式推导，引用案例包括：
- 煤矿动力灾害风险评估（Zhang2023）
- 矿区治理模型选择（Liang2023）
- 农业气象灾害风险区划（Chen2016）
- 城市土地集约利用（Shang2009）
- 旅游地社区参与度（Hou2010）

**所有这些文献没有一个与奥运或体育评估有关。** 读者完全无法理解：为什么煤矿风险评估的方法适用于奥运项目筛选？不加论证地将来自完全不同领域的方法直接移植，构成方法论层面的逻辑断裂。

### 3. EWM权重完全受项目选择偏差支配，论文对此只字不提

熵权法的权重取决于所选样本的指标变异程度。论文选了12个项目（4核心+4新增+2候选+2已移除），人为构造了类别间的最大差异。如果这12个项目选得不同，EWM权重将完全不同。论文从未讨论项目选择对EWM结果的决定性影响，反而将权重差异包装为"客观数据揭示的规律"。这不是客观，这是**用刻意筛选的样本制造预设结论**。

### 4. 文献综述是文献堆砌而非述评

`thesis.tex:194-240` 的"国内外研究现状"呈现典型的"XXX说……YYY说……"罗列模式：
> Saaty提出AHP → Wang提出EWM → Zhang用于煤矿 → Liang用于矿区 → Chen用于农业 → Shang用于土地 → Hou用于旅游

七个引用平铺直叙，零分类、零归纳、零批判。作者始终未回答一个根本问题：**前人研究的不足到底是什么？本研究填补的空白在哪里？** 连"研究空白"都无法指认的文献综述，不具备任何学术价值。

### 5. AHP判断矩阵在不同文件中互相矛盾

- `thesis.tex:666-674` 公布的综合判断矩阵，元素如 `(1,2)=1.644`、`(2,1)=0.608`

两者差异达+62%和-38%，绝非同一矩阵。图表与正文互相矛盾。

### 6. 数据管道计算结果与绘图硬编码值完全脱节

以**电子竞技**为例：

## 三、结构与逻辑性批判

### 1. 标题三大关键词权重失衡，"系统设计"名不副实

标题中有三个核心词：评估、预测、系统设计。第六章"系统设计与实现"仅约80行（`thesis.tex:1356-1435`），占全文总篇幅不足5%，几乎没有任何实质性的系统架构细节——没有技术选型说明、没有模块接口定义、没有测试用例、没有用户评估。如果论文标题包含"系统设计"，这一章不应比致谢还短。

### 2. 绪论与结论无逻辑闭环

绪论提出研究问题："当前奥运项目的评估主要依赖专家委员会的主观判断，缺乏系统化的量化工具"（`thesis.tex:148`）。结论应回答：该问题是否被解决？如何解决？遗留什么？但第六章总结（`thesis.tex:1447-1470`）是第五章内容按编号重述了一遍（"第一……第二……第三……第四……第五"），没有任何高于正文的提炼与归纳。绪论的提问在结论中杳无踪迹。

### 3. "现状-问题-对策"三段论机械套用

从绪论（背景→意义→现状→方法）、第二章（理论概述→AHP→EWM→混合→奥运）、第三章（数学建模→指标体系→AHP→EWM→混合→有效性），到第四章（数据→实验→对比→灵敏度→预测→讨论），每一章内部都采用同样的"引入→展开→小结"模板。整篇论文读起来像五个平行排列的填空题模板，而非层层递进的学术论证。

### 4. 各章小结是正文的废话复读机

第三章小结："本章构建了一个基于AHP和EWM的混合权重评估模型。先把……然后……最后……"（`thesis.tex:931-935`）。  
第四章小结："本章利用HiMCM大赛……对第三章构建的……进行了实验验证。具体工作包括……"（`thesis.tex:1341-1353`）。

每一章小结就是前文的缩略版，毫无提炼。这是典型的凑字策略。

### 5. 摘要包含研究背景、方法、核心结论？——不，它只是引言的缩写

摘要中"首先……其次……然后……"的结构，与第一章研究内容的编号列举几乎完全一致。摘要没有独立提炼任何高于正文的信息，只是将引言和目录又复述了一遍。

## 四、细节与写作规范

### 1. 口语化与不规范性语句逐句批注（截取自 `thesis.tex:563-575, 703-713`）

> "先把问题用数学语言描述一下。"

**批注：** "先……一下"是典型的口语表达。学术写作应为"将评估问题形式化为多准则决策模型"。

> "权重怎么定是整个模型的关键。"

**批注：** "怎么定"是口语，"整个模型的关键"是空洞判断。应表述为"权重向量的确定方法直接决定评估结果的有效性"。

> "看第一行的话，流行度跟性别平等比是1.644，跟可持续性比是3.677，跟包容性比是4.338，跟创新性比是5.404，跟安全性比是2.000。这说明流行度被认为是最重要的……"

**批注：** "看第一行的话"是口述稿式表述。逐一念出矩阵元素值属于无意义的数据复读——读者有眼睛，不需要你代为朗读判断矩阵。

> "算出来……一致性挺好。"

**批注：** "算出来"是口语，"挺好"是主观评价词。学术表述应为"计算得 CR=0.003<0.1，通过一致性检验"——结论让数据说话，不需要"挺好"。

> "这个分布跟实际情况差不多，IOC最看重项目的影响力和普及度……"

**批注：** "差不多"是模糊口语。如果IOC确实最看重流行度，请引用IOC官方文件证明；如果没有引用而出此断言，属于无据推测。

## 五、Python 脚本准确性专项评判

### 1. `experts_data.py` —— 专家数据伪造脚本

这不是"数据处理"脚本，而是**数据造假工具**。

**关键代码（第24-36行）：**

```python
def perturb_matrix(M, noise_std=0.20):
    ...
    elif i < j:
        noise = np.random.lognormal(0, noise_std)   # ← 对数正态噪声
        val = M[i, j] * noise
        Mp[i, j] = round_to_saaty(val)              # ← 四舍五入到萨蒂标度
        Mp[j, i] = 1.0 / Mp[i, j]
```

第53-62行设定了五位专家的"真实"权重向量，然后据此构造"完全一致矩阵"，再人为添加噪声、四舍五入到1-9标度。第158行更是直接暴露问题：

```python
orig_weights = [0.391, 0.189, 0.106, 0.092, 0.060, 0.163]
```

标注为"原论文AHP"，说明脚本作者**明知**自己生成的数据与论文中声称的数据不一致，却仍将其作为"专家数据"输出。

### 2. `generate_figures.py` —— 核心数学计算错误

#### （1）AHP 判断矩阵与论文严重不符

`generate_figures.py` 第18-25行硬编码的判断矩阵，与论文 `thesis.tex:666-674` 公布的矩阵差异达62%以上。使用不同矩阵计算出的AHP权重必然不同，图表与正文互相矛盾。

#### （2）EWM 计算中的零值处理扭曲信息熵

```python
p = Xn / Xn.sum(axis=0, keepdims=True)
p = np.where(p==0, 1e-10, p)
```

当某个项目在某维度的归一化值为0时，`p_ij * ln(p_ij)` 的严格极限为0。但脚本人为将其替换为 `1e-10`，改变了熵的严格定义，累积误差。

#### （3）综合评分使用归一化矩阵而非原始矩阵

第108行：`scores = Xn @ Wh`。论文声称六维评分 `E_ij` 来自子指标的EWM加权（两层结构）——脚本中完全没有第一层（子指标→维度）的计算。12个项目的六维评分全部是硬编码的。

### 3. `generate_enhanced_figures.py` —— 硬编码数据与分类错误

#### （1）硬编码评分矩阵与 `data_pipeline.py` 计算结果完全脱节

以电子竞技为例：
- `data_pipeline.py` 计算：可持续性 ≈ 0.5，创新性 ≈ 0.5
- 绘图脚本硬编码：可持续性 0.85，创新性 1.00

以板球为例：
- `data_pipeline.py` 计算：流行度 ≈ 0.072，创新性 ≈ 0.023
- 绘图脚本硬编码：流行度 0.85，创新性 0.45

#### （2）电子竞技被错误标记为"核心"项目

第38行：
```python
CATEGORIES = np.array(["核心","核心","核心","核心","核心", ...])
```

对应 `NAMES` 第2个元素是"电子竞技"，被标记为"核心"。但论文明确将电子竞技归为"候选项目"。这导致散点图和分类对比图中电子竞技被错误着色，**图表传达的分类信息与论文文字直接矛盾。**

#### （3）混合权重未归一化

第21行：
```python
Wh = 0.5 * Wa + 0.5 * We
```

虽然和接近1.0，但未执行 `Wh /= Wh.sum()`。而 `generate_figures.py` 第101行做了归一化。两个脚本标准不一致。

#### （4）EWM 权重直接硬编码

第20行：
```python
We = np.array([0.218, 0.100, 0.108, 0.174, 0.297, 0.104])
```

EWM权重应从12个项目的六维评分数据中计算得出，但脚本直接硬编码了论文中的EWM结果，跳过了全部计算过程。

### 4. `data_pipeline.py` —— 指标公式中的任意性与逻辑缺陷

#### （1）流行度公式中的无依据常数

```python
events_ratio = min(total_events / max(total_appearances, 1) / 50, 1.0)   # ← 除以50的依据？
recency_score = min(recent / max(total_events, 1) * 3, 1.0)             # ← 乘以3的依据？
score = 0.35 * appearances_ratio + 0.25 * events_ratio + 0.2 * growth_score + 0.2 * recency_score  # ← 权重系数依据？
```

论文中对这些任意常数没有任何理论说明。它们的存在使"客观计算"变成了"主观调参"。

#### （2）可持续性公式中的概念混淆

```python
score = 0.4 * continuity + 0.35 * stability + 0.25 * (1 - resilience * 2)
```

其中 `resilience` 被定义为"被移除后又回来的次数"。棒球/垒球曾在2008年被移除、2020年回归，存在gap。按此公式，gap越多，`resilience`越高，可持续性得分越低。也就是说，**曾经被移除但成功回归的项目，反而被惩罚为"不可持续"**。这与"resilience（韧性）"一词的常规语义相反，属于概念混淆。

#### （3）包容性公式中的时代划分随意性

```python
eras = [(1896, 1920), (1924, 1952), (1956, 1984), (1988, 2024)]
```

四个时代的划分是**完全随意的**，没有任何体育史或奥运史的理论支撑。为什么是4个时代？为什么边界是1920、1952、1984？论文对此只字未提。

### 5. `data_validate.py` —— 验证流于形式

该脚本仅检查：
1. 数值是否在 [0, 1] 区间内
2. 是否存在缺失值

**未检查：** 数据与原始Excel的一致性、指标计算公式是否正确执行、归一化前的原始值是否合理、项目分类是否与论文一致。一个数据验证脚本完全不验证数据来源和计算逻辑，等于形同虚设。

### 6. 跨文件一致性总览

| 数据项 | 论文正文 | `experts_data.py` | `generate_figures.py` | `generate_enhanced_figures.py` | `data_pipeline.py` |
