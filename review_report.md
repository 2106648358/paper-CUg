# 本科毕业论文《基于混合权重模型的奥运项目评估与预测系统设计》评审报告

**评审人**：AI 学术评审

## 总体评价

论文选题具有现实意义，将 AHP-EWM 混合权重模型引入奥运项目评估领域，构建了六维评价指标体系，并完成了模型构建、实验验证和系统实现。工作量大，结构完整。但在内容一致性、数据严谨性、方法论证深度方面存在若干必须修正的问题。

---

## 一、严重问题（必须修正）

### 1.1 事实性矛盾：第六章总结与其他章节结论不符

**位置**：thesis.tex:1589-1590 (`\subsection{研究总结}`)

> "电子竞技排名第一，反映了其在流行度和创新性方面的优势。"

**问题**：第四章实验结果显示田径排名第1（0.748），电子竞技排名第2（0.743）。该总结与正文数据直接矛盾，属于严重的事实性错误。中文摘要（行51）表述的是正确的（"田径以0.748分位列第一，电子竞技以0.743分紧随其后"），说明同一结论在不同章节出现了一致性问题。

**建议**：全文统一为"田径第一、电子竞技紧随其后"的表述。

### 1.2 方法对比节权重数值多处不一致

**位置**：thesis.tex:1262-1292 (`\subsection{方法对比}`)

以下数值与第三章的表格数据严重不符：

| 指标 | 4.3节所述EWM值 | 表3.4实际EWM值 | 差异 |
|------|----------------|----------------|------|
| 流行度 | 0.218 (行1285) | 0.144 | +0.074 |
| 创新性 | 0.297 (行1285) | 0.198 | +0.099 |
| 流行度(纯AHP排名) | 0.290 (行1270) | — | 无法对应 |

**问题**：
1. 行1267-1268称"创新性因数据变异程度最大而被赋予最高权重"，但表3.4显示可持续性的EWM权重最高（0.272），创新性仅为0.198。
2. 行1270"流行度权重从0.362降至0.290"中的0.290、行1271"创新性权重从0.297降至0.181"中的0.297和0.181，与表3.4、表3.6的数值均不匹配。

**可能原因**：4.3节的权重数据可能来源于另一个版本的EWM计算（如仅对核心项目的子集计算），或是在成文过程中数值被修改后未同步到相关章节描述。**全文必须统一所有权重数值，确保文本描述与表格数据一致。**

### 1.3 EWM样本量过小且未充分论证

**位置**：thesis.tex:453-462, 1454-1459

EWM权重的计算仅基于12个样本（m=12），而指标数为6个。从统计角度看：
- 12个样本的信息熵估计方差较大，EWM权重对个别项目的增减非常敏感。
- 作者虽然在第3.4.3节（行453-462）和4.6节（行1453-1459）讨论了EWM对样本构成的依赖性，但未给出定量论证（如Bootstrap方法估计权重置信区间）。

**建议**：增加量化分析，如通过Bootstrap计算各维度EWM权重的置信区间，或通过留一法验证权重稳定性。

## 二、较重要问题

### 2.1 AHP五位"专家"为模拟专家，论文未明确说明

**位置**：thesis.tex:708-735

五张判断矩阵是由作者根据设定的"权重偏好"反推生成的，而非来自真实专家调研。表3.2中的权重各有不同，似乎是为了增强对比性而设计。论文正文表述"本研究邀请了五位不同背景的领域专家"（行708）容易让读者误以为是真实的专家调研。

**建议**：
- 将"邀请了"改为"设定了"或"模拟了"，明确说明此为模拟实验。
- 或将此作为论文重要局限性在第六章中加以说明。
- 如果能补充至少2-3位真实领域专家的判断矩阵，将大幅提升论文的信度。

### 2.2 论文冗余度高，第二、三章内容大量重复

- AHP的1-9标度法、判断矩阵、特征值求解、一致性检验在2.2节和3.3节各讲了一遍。
- EWM的标准化公式（式2.8-2.9 与 式3.2-3.3）几乎完全相同。
- 信息熵公式（式2.10 与 式3.7-3.8）重复出现。
- 混合权重的线性组合公式（式2.13 与 式3.9）也重复了。

**建议**：
- 第二章应专注于文献综述（前人做了什么、有什么不足），不应展开推导完整公式。
- 所有数学模型和公式推导集中到第三章。
- 第二章中的方法对比表（表2.3）保留，但公式部分精简为引用即可。

### 2.3 第五章（系统实现）与研究主体关联薄弱

全文共六章，第五章描述了一个React+TypeScript的Web评估系统。当前写法更像是软件技术文档，缺乏与研究成果的学术关联论证：
- 未说明该系统的开发如何验证了模型的有效性。
- 未讨论系统的实际使用效果（如用户测试反馈）。
- 作为一个本科毕业论文，描述UI组件和React架构的意义有限。

**建议**：缩短第五章篇幅，将重点放在"系统如何体现模型价值"而非"技术实现细节"上，或者将第五章内容移至附录。

### 2.4 安全性维度数据引用的Edouard2026年份存疑

**位置**：references.bib:211-220, thesis.tex:145

`Edouard2026` 标注年份为2026年，卷号为60（第6期）。当前日期为2026年5月，该期刊是否能在此时间点出版到第60卷第6期存疑。建议核实该文献的实际发表年份，是否存在DOI提前发布的情况。如果是2025年online-first而2026年才会正式出版，应在bib条目中注明。

## 三、一般性问题

### 3.1 英语摘要语言问题

- 关键词 "AHP-EWM hybrid weight" 与 "entropy weight method" 存在语义重叠。
- "Olympic sport evaluation" 应为 "Olympic sports evaluation"。
- 第87行 "forming a two-layer structure" 句子不完整，缺少主句。
- 多处句子较长，建议拆分为短句以提升可读性。

### 3.2 引用格式问题

- `IOC2023`、`IOC2024`、`IPC2024` 在 bib 文件中标记为 `@article`，实际均为网页来源，应使用 `@online` 或 `@misc` 类型，并标注访问日期。
- `Miller1956` 在 bib 中定义了但正文未引用。第332-334行提到"7±2个等级"但未给出引用。
- `Hunter2007` (matplotlib) 在 bib 中定义了但正文未引用。
- `HiMCM2024` 在 bib 中定义了但未在正文中明确引用。

### 3.3 LaTeX排版问题

- **行4**：`\usepackage{...}` 放在 `\title{...}` 等元数据声明之间，应全部移至document环境之前、元数据声明之后。
- **行14**：`\usepackage{makecell,multicol,...}` 中 `subcaption` 包已被 CUGThesis.cls 加载（行28），重复加载可能导致选项冲突。
- **行121, 285, 568, 1044, 1478, 1565, 1678**：全文使用 `\begin{spacing}{2}\section{...}\end{spacing}` 的方式设置章节标题行距，非规范做法。应在导言区用 `\usepackage{setspace}`（已加载）配合 cls 文件设置全局行距。
- **全文多处**：figure 环境使用 `[H]` 位置参数，可能导致大量留白。建议对非关键图表使用 `[htbp]`。
- **行550**：`\begin{tabularx}{\textwidth}{lcc>{\raggedright\arraybackslash}X}`（方法对比表），由于前三列为居中或左对齐，可以利用 `X` 列的自动宽度以更好利用空间。

### 3.4 数据处理与方法论细节缺失

- 行1084-1086：HiMCM数据与IF会员国数据以0.70:0.30混合——为什么选择这个比例？缺乏论证。
- 行1032（算法3，行27）：`W = W / ΣW` 这步归一化对凸组合而言是冗余的（AHP和EWM权重均已归一化，线性凸组合自动保持和为1）。
- 行996（算法1）：伪代码的 return 语句在 `\IF{...}\RETURN...\ELSE\RETURN...\ENDIF` 结构中，逻辑正确但建议在ELSE分支也返回一个值或错误标识。
- 灵敏度分析仅展示了α变化的结果，未涉及判断矩阵扰动（AHP灵敏度）或决策矩阵扰动（EWM灵敏度）。

### 3.5 表述问题

- 行1294-1295 解释"雷达图展示的是评分而非权重"——这段解释本身是对的，但行数过长（占4行），且前面已经多次区分了评分和权重的概念。
- 行1715 "同窗之谊是我大学生活中珍贵的收获"——作为致谢中的抒情感慨，这一句适宜。
- 行1707 "部分已引用文献的DOI因格式错误无法解析"——说明文献核查工作较为认真。
- 全文口语化表达较多（如"比如"、"差不多"、"好处在于"），建议在正式论文中替换为更学术的措辞。

### 3.6 图与表的编号

全文图/表编号为独立体系（按章节编号），这一点是规范的。但有少量表格和图片引用在文中的顺序略有问题：
- 图4.5（权重差异，line 1287）在行1285-1286的文本后引用，但图实际上在行1287-1292，尚无问题。
- 图4.4（方法排名对比，line 1276）在文本描述之前或之中？需确认引用的位置与图表实际出现位置的对应关系符合"先出现引用，再出现图表"的排版惯例。

## 四、肯定之处

1. **选题切合热点**：奥运项目选取是IOC当前核心议题之一，量化评估工具具有实际需求。
2. **数据工作量充足**：从HiMCM原始数据（76个项目×32届）到12个代表性项目的六维评分，再到外部数据补充（受伤率文献、IF会员国数据），数据处理流程完整。
3. **方法论逻辑清晰**：AHP提供主观权重、EWM提供客观权重、线性组合混合——三阶段的方法设计思路清晰，且对每一步的动机和优缺点有分析。
4. **灵敏度分析意识**：虽然方法简单，但对α系数的扫描分析以及对EWM样本依赖性的讨论体现了作者对方法局限性的认知。
5. **中英文双语摘要**：格式规范，能独立反映全文要点。
6. **致谢部分真实具体**：文献DOI核查、数据预处理困难等细节描述真实可信，体现了作者的研究态度。
